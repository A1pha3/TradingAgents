# 多语言输出 ⭐

> **目标读者**：想让报告输出中文或其他语言的使用者
> **预计时间**：5 分钟
> **前置知识**：[CLI 交互式手册](cli-manual.md) ⭐⭐

---

## 一句话说明

TradingAgents 支持让分析师报告和最终决策输出 **11 种语言**，配置方式有两种：CLI 交互选择，或环境变量固定。

---

## 两种配置方式

### 方式一：CLI 交互选择

在 CLI 步骤 3 选择输出语言：

```
Step 3: Output Language
> 中文
```

支持的语言（`cli/utils.py:653-688`）：

| 语言 | 选项名 |
|------|--------|
| 英语 | English |
| 简体中文 | 中文 |
| 西班牙语 | Español |
| 法语 | Français |
| 德语 | Deutsch |
| 日语 | 日本語 |
| 韩语 | 한국어 |
| 俄语 | Русский |
| 葡萄牙语 | Português |
| 意大利语 | Italiano |
| 荷兰语 | Nederlands |

另外有一个 `Custom` 选项，可以输入任意语言名称（比如"阿拉伯语"）。

### 方式二：环境变量固定

如果你不想每次都选，用环境变量一劳永逸：

```env
TRADINGAGENTS_OUTPUT_LANGUAGE=中文
```

写入 `.env` 文件后，CLI 的步骤 3 会自动跳过。这个环境变量在 `DEFAULT_CONFIG` 构造时就被应用，等价于把默认值改成中文。

---

## 一个关键细节：内部辩论保持英文

这是理解 TradingAgents 多语言机制最重要的一点：**输出语言只影响分析师报告和最终决策的呈现语言，不影响 Agent 之间的内部辩论。**

```
                      output_language 的作用范围
                    ┌─────────────────────────────┐
                    │                             │
  分析师报告         │  ← 受 output_language 影响   │
  最终决策           │  ← 受 output_language 影响   │
                    │                             │
  Bull/Bear 辩论     │  ← 始终英文，不受影响        │
  风险辩论           │  ← 始终英文，不受影响        │
  Tool 调用参数      │  ← 始终英文，不受影响        │
                    │                             │
                    └─────────────────────────────┘
```

### 为什么这样设计

这是基于 LLM 推理质量的工程权衡。当前主流 LLM（GPT、Claude、Gemini）的预训练语料以英文为主，在英文 prompt 下推理质量最稳定。辩论环节是框架质量的核心——多空双方的逻辑推理、证据权衡、反驳质量直接决定最终决策的好坏。

如果把辩论也切成中文，会引入两个风险：一是 LLM 在非英文下的推理能力下降，二是某些专业金融术语的中文翻译可能产生歧义。框架选择让辩论保持英文、只在最终产出层做语言切换，是用最小的成本换取最大的质量保证。

技术实现上，语言指令通过 `get_language_instruction`（`agent_utils.py:52-65`）注入 prompt。当 `output_language` 是 English 时返回空串省 Token；非 English 时返回类似 "Write your report in 中文." 的指令。分析师和组合经理会收到这个指令，辩论角色不会。

## 验证语言设置生效

运行分析后，检查报告里的语言：

```bash
# 查看最终决策
cat ~/.tradingagents/logs/reports/NVDA_*/5_portfolio/decision.md | head -20
```

如果开头是中文摘要，说明设置生效。

## 常见问题

### Q：设置了中文，但报告里混了英文术语

这是正常现象。金融领域的专有名词（PE ratio、MACD、Bollinger Bands 等）在中文报告里保留英文是常见做法，因为它们的中文翻译不统一。框架不会强行翻译这些术语。

### Q：不同语言会影响分析质量吗

不会影响分析逻辑本身（辩论保持英文），只影响最终报告的可读性。但如果你用非英文的分析师报告做下游处理，需要注意分词和 NLP 工具的语言兼容性。

### Q：可以输出英文报告但用中文做内部处理吗

不行，设计恰好相反：内部处理用英文（保证质量），输出层切换语言（保证可读）。这是经过权衡的设计，不可配置为反向。
